Round 2 — reachability & wake

Variant C (the unified worklist) shipped. Two problems remain. (1) Offline isn't distinguishable — the offline tag uses the attn tone \x1b[1;33m, which is byte-identical to accent, the selection-marker gold. An offline row that needs you looks exactly like a selected/needs-you row. (2) Entering an offline agent is janky — Enter silently routes through a resume (it wakes the tmux session, a slow op) while the screen sits on Coordination giving no feedback; the row never told you Enter meant wake, not open. These variants fix both: a dormant sleep tone distinct from the accent, and an explicit wake verb so the action is honest.

Variant D — Reachability dot + wake verb (recommended)

Smallest change on top of shipped C; fixes both problems. The leading dot encodes reachability, not just actionability: ● awake · ◐ asleep · ○ gone. The reachability tag moves to a dormant sleep tone (--sleep, a cool slate) that can never be confused with the gold accent. The contextual footer shows the true verb for the selected row — open for awake, wake for asleep (slow resume), and — (disabled) for gone — so Enter is honest and the resume is expected, not a surprise. The detail card spells out what waking does.

Variant F — Reachability-split buckets

Make reachability structural instead of per-row: split "Needs you" into Awake · act now and Asleep · wake to act. Offline agents physically separate from ones you can touch instantly, so the jank disappears — you never expect an instant open from the Asleep group. Strongest signal, biggest structural change; risks a near-empty Asleep group most of the time.
