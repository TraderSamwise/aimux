# aimux Agent Instructions

You are running inside aimux, an agent multiplexer for this repository. Aimux keeps long-lived Claude, Codex, and shell sessions in the main checkout and git worktrees so the user can switch between them, stop/restart them, and coordinate work.

## Aimux Model
- The user controls aimux from the dashboard and tmux status/footer UI.
- Agents are normal tool processes running inside aimux-managed tmux windows.
- Cross-agent coordination is file-based: use `.aimux/tasks/*.json` when explicitly asked to delegate or hand off work.
- Do not assume you can directly invoke another live agent unless the user gives a specific aimux CLI command or asks you to create an aimux task.

## Shared Context Files
- `.aimux/context/{session-id}/live.md` — recent conversation for this session
- `.aimux/context/{session-id}/summary.md` — compacted history for this session
- `.aimux/plans/{session-id}.md` — optional shared plan for long-running or delegated work
- `.aimux/status/{session-id}.md` — optional brief status note for long-running or delegated work
- `.aimux/tasks/{task-id}.json` — explicit handoff records when the user asks for delegation
- `.aimux/context/{other-session-id}/` — other agents' context when needed
- `.aimux/history/` — full raw conversation history (JSONL)

Do not proactively create or edit `.aimux/plans/*` or `.aimux/status/*` for simple questions, read-only inspections, or one-shot tasks. Only update those files when the user asks for coordination/delegation, when the task is explicitly long-running, or when state would materially help another agent continue the work.

## Delegation Protocol
When asked to delegate, hand off, or assign work to another aimux agent, create `.aimux/tasks/{task-id}.json` with `id`, `status: "pending"`, `assignedBy`, `description`, `prompt`, `createdAt`, and `updatedAt` using ISO timestamps. Optional fields are `assignedTo` for a specific session ID and `tool` for coordination metadata. Treat task files as the shared handoff record; do not call aimux metadata APIs from inside an agent unless the user gives an explicit CLI/API command.
When you accept a task file, complete it and mark the task `done` with `result`, or `failed` with `error`.

This file is auto-generated by aimux and will be removed when aimux exits.
